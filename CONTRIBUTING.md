# Contributing to AI File Analyzer

Thanks for your interest! This is a personal project, but I'm open to contributions.

## How to Contribute

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/cool-feature`)
3. Make your changes
4. Test everything
5. Commit with a descriptive message
6. Push and create a PR

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/ai-file-analyzer.git

# Install dependencies
npm install

# Copy env file and add your API key
cp .env.example .env.local

# Run dev server
npm run dev
```

## Code Style

- Use TypeScript
- Follow existing patterns
- Keep components small and focused
- Add comments for complex logic
- Use meaningful variable names

## What I'm Looking For

- Bug fixes
- Performance improvements
- New file format support
- UI/UX improvements
- Better error handling
- Documentation improvements

## What I'm NOT Looking For

- Major architectural changes (discuss first)
- Breaking changes without good reason
- Unnecessary dependencies
- Code that doesn't follow the existing style

## Testing

Make sure to test:
- Different file types
- All language options
- All analysis detail levels
- Mobile responsiveness
- Error cases
